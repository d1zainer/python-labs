# Кейс №2_1 --- Пояснения к коду

## Генерация данных

В этом блоке создаётся искусственный датасет с помощью NumPy.

-   `np.random.seed(7)` фиксирует генератор случайных чисел для
    воспроизводимости результатов.
-   `course_id` --- случайное распределение трёх курсов.
-   `views` --- случайное количество просмотров страницы курса.
-   `enrollments` --- 8% от просмотров (модель конверсии).
-   `costs` --- затраты на продвижение (случайное значение от 50 до 150
    руб. за запись).
-   `completions` --- 60% от записавшихся (модель доходимости).

Данные объединяются через `np.column_stack()` и сохраняются в CSV-файл.

Цель блока --- создать тестовый набор данных для дальнейшего анализа.

## Загрузка данных

Данные загружаются из CSV-файла с помощью `np.loadtxt()`.

Параметры: - `delimiter=','` --- разделитель столбцов - `skiprows=1` ---
пропуск строки заголовка

Проверяется корректность загрузки через вывод первых строк и формы
массива.

## Выделение столбцов

Через срезы массива выделяются отдельные столбцы:

-   `course_id`
-   `views`
-   `enrollments`
-   `costs`
-   `completions`

Каждый столбец сохраняется в отдельный массив для удобства вычислений.

## Векторизация

Векторизация --- выполнение операций сразу над всем массивом без
использования циклов.

Формулы:

CR = (Enrollments / Views) \* 100\
CPC = Costs / Completions

Используется `np.where()` для защиты от деления на ноль:

-   если знаменатель \> 0 → выполняем расчёт
-   иначе → записываем `np.nan`

Результат:

-   `cr` --- массив конверсии по каждой строке
-   `cpc` --- массив стоимости завершения по каждой строке

Мы получаем не одну формулу, а массив результатов её применения ко всем
данным.

## Расчёт статистики

Используются функции:

-   `np.nanmean()` --- среднее значение без учёта NaN
-   `np.nanmin()` --- минимальное значение
-   `np.nanmax()` --- максимальное значение

Это позволяет корректно анализировать данные даже при наличии пропусков.

## Группировка по курсам

`np.unique(course_id)` возвращает список уникальных курсов.

Для каждого курса:

1.  Создаётся маска: `mask = course_id == cid`
2.  Через маску фильтруются строки конкретного курса
3.  Считаются суммарные затраты и завершения

Получаем:

-   `total_cost` --- общие затраты по каждому курсу
-   `total_comp` --- общее число завершений по каждому курсу

## Расчёт эффективности курса

Метрика эффективности:

Стоимость завершения = Общие затраты / Общее число завершений

Используется `np.where()` для защиты от деления на ноль.

`course_eff` --- массив эффективности каждого курса.

## Поиск топ-2 курсов

`np.argsort(course_eff)` возвращает индексы курсов в порядке возрастания
стоимости.

Берутся первые два индекса --- это курсы с минимальной стоимостью
завершения (самые эффективные).

Результат выводится через цикл.
